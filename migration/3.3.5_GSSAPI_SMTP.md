# v3.3.5 — GSSAPI (Kerberos 5) Authentication for SMTP

## What Changed

GSSAPI/Kerberos 5 authentication (SASL GSSAPI, RFC 4752) is now supported for the `gomailtest smtp` commands (`testauth`, `sendmail`). This enables testing on-premises Exchange and Active Directory SMTP environments that mandate Kerberos over NTLM.

**New dependency:** `github.com/jcmturner/gokrb5/v8 v8.4.4` — pure Go, cross-platform, no CGO.

## Username and Realm Formats

| Format | Example | Realm |
|--------|---------|-------|
| `user@REALM.COM` | `alice@CONTOSO.COM` | `CONTOSO.COM` (from `@` suffix) |
| `DOMAIN\user` | `CONTOSO\alice` | `CONTOSO` (from prefix) |
| plain | `alice` | requires `--realm` flag |

Realm is always uppercased automatically. The `--realm` flag overrides auto-detection.

## KDC Discovery

Without `--kdc`, the KDC is discovered via DNS SRV records (`_kerberos._tcp.<realm>`), which is the standard AD configuration.

Use `--kdc dc01.contoso.com` (or `dc01.contoso.com:88`) to override when DNS SRV is not configured or to target a specific domain controller.

## Auto-Detection Priority Change

When `--authmethod auto` (default), the new priority order is:

**Without access token:** **`GSSAPI`** → `CRAM-MD5` → `NTLM` → `PLAIN` → `LOGIN`

**With `--accesstoken`:** `XOAUTH2` → **`GSSAPI`** → `CRAM-MD5` → `NTLM` → `PLAIN` → `LOGIN`

GSSAPI is only selected when the server explicitly advertises it in the EHLO `AUTH` capability list.

## New Flags

| Flag | Env var | Description |
|------|---------|-------------|
| `--realm` | `SMTPREALM` | Kerberos realm override |
| `--kdc` | `SMTPKDC` | KDC address override (host or host:port) |

## Example Usage

```powershell
# Explicit GSSAPI — realm extracted from username
gomailtest smtp testauth --host exchange.contoso.com --port 25 \
  --username "alice@CONTOSO.COM" --password "secret" --authmethod GSSAPI

# With explicit KDC (when DNS SRV not configured)
gomailtest smtp testauth --host exchange.contoso.com --port 25 \
  --username "alice@CONTOSO.COM" --password "secret" --authmethod GSSAPI \
  --kdc dc01.contoso.com

# Auto-detect (picks GSSAPI when server advertises it)
gomailtest smtp testauth --host exchange.contoso.com --port 25 \
  --username "alice@CONTOSO.COM" --password "secret"

# Send via GSSAPI-authenticated relay
gomailtest smtp sendmail --host exchange.contoso.com --port 25 \
  --username "alice@CONTOSO.COM" --password "secret" \
  --from alice@contoso.com --to bob@contoso.com \
  --subject "Test" --body "GSSAPI relay test"
```
