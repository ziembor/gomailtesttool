**Branch:** `b3.0.3`

- `internal/protocols/msgraph/cert_stub.go` ‚Äî Non-Windows cert store stub (package rename)
- `internal/protocols/msgraph/cert_windows.go` ‚Äî Windows cert store (package rename)
- `internal/protocols/msgraph/utils.go` ‚Äî Logging helpers, maskSecret, retry wrappers (package rename)
- `internal/protocols/msgraph/utils_test.go` ‚Äî Unit tests (package rename; tests now in `msgraphtool/internal/protocols/msgraph`)
- `internal/protocols/msgraph/handlers.go` ‚Äî All 7 handler functions; `executeAction()` switch removed (replaced by Cobra subcommand routing)
- `internal/protocols/msgraph/config.go` ‚Äî `Config` struct, `NewConfig()`, action constants, `validateConfiguration()`, `stringSlice`, Viper-based `RegisterPersistentFlags()`, `BindEnvs()`, `ConfigFromViper()`
- `internal/protocols/msgraph/cmd.go` ‚Äî `NewCmd()` exposing `msgraph` Cobra command with 7 subcommand leaves (getevents, sendmail, sendinvite, getinbox, getschedule, exportinbox, searchandexport)
- `cmd/gomailtest/main.go` ‚Äî 10-line entry point calling `Execute()`
- `cmd/gomailtest/root.go` ‚Äî Root `gomailtest` Cobra command; registers `msgraph.NewCmd()`

**Modified files:**
- `cmd/msgraphtool/main.go` ‚Äî Replaced with exec-based backward-compat shim; translates old `-action=X -flagname=Y` style to new `msgraph X --flagname=Y` style, locates `gomailtest` binary in same directory or PATH
- `go.mod` ‚Äî Promoted `cobra` and `viper` from indirect to direct dependencies; `go mod tidy` run
- `internal/common/version/version.go` ‚Äî Bumped to `3.0.3`

**Deleted files:**
- `cmd/msgraphtool/{auth,cert_stub,cert_windows,config,handlers,utils,utils_test}.go` ‚Äî All moved to `internal/protocols/msgraph/`

**Verification performed:**
- `go build ./...` clean
- `go test ./...` ‚Äî all packages pass; `internal/protocols/msgraph` tests run 1.8s
- `gomailtest msgraph --help` shows all 7 subcommands with correct flags
- Both `gomailtest.exe` and `msgraphtool.exe` (shim) build successfully

**Key design decisions made during execution:**
- **Shim uses `exec.Command`** (not in-process dispatch) ‚Äî `cmd/gomailtest/` stays `package main`, no restructuring needed. Shim finds `gomailtest` in same dir first, then PATH.
- **Viper instance is per-protocol** (created in `msgraph.NewCmd()`), not global ‚Äî prevents flag name collisions when Unit D adds other protocols.
- **Signal context in each subcommand's RunE** via `bootstrap.SetupSignalContext()`, not in rootCmd's `PersistentPreRunE` ‚Äî keeps each action self-contained.
- **`--verbose` flag** is on `msgraph` persistent flags (not root), since other protocols may have different verbosity implementations.
- `stringSlice.Type()` method added to satisfy `pflag.Value` interface (required for Cobra flag registration).
