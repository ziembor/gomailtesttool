# gomailtesttool 3.2 — EWS Protocol Support

This document outlines the implementation plan for adding Exchange Web Services (EWS) as a new protocol subcommand in `gomailtest`.

## Goal

Add `gomailtest ews` to cover on-premises Exchange Server (2007–2019) diagnostics via the EWS SOAP/HTTP API (`/EWS/Exchange.asmx`). This fills the gap that `gomailtest msgraph` cannot cover — MS Graph is cloud-only (Exchange Online), while EWS targets on-premises deployments.

---

## New Dependency

```
github.com/Azure/go-ntlmssp
```

Provides NTLM transport negotiation used by the HTTP EWS client. Add via:

```
go get github.com/Azure/go-ntlmssp@latest
```

---

## Command Structure

```
gomailtest ews
  ├── testconnect    — HTTP/TLS connectivity check (no credentials required)
  ├── testauth       — Authenticate via NTLM, Basic, or Bearer token
  ├── getfolder      — GetFolder(Inbox): display name, total/unread count, folder ID
  └── autodiscover   — Test SOAP Autodiscover endpoint, report resolved EWS URLs
```

**Authentication methods:** `NTLM` (on-premises AD), `Basic`, `Bearer` (OAuth2 for Exchange Online EWS), `auto`

**Environment variable prefix:** `EWS` (e.g. `EWSHOST`, `EWSUSERNAME`, `EWSPASSWORD`, `EWSAUTHMETHOD`, `EWSMAILBOX`)

## Files to Create

| File | Purpose |
|---|---|
| `internal/protocols/ews/config.go` | Config struct, flags, `BindEnvs`, `ConfigFromViper`, `validateConfiguration` |
| `internal/protocols/ews/ews_client.go` | HTTP/SOAP EWS client: NewEWSClient, auth setup, TLS config |
| `internal/protocols/ews/cmd.go` | `NewCmd()` + 4 subcommand constructors following `smtp/cmd.go` pattern |
| `internal/protocols/ews/testconnect.go` | HTTP/TLS probe — 401 is acceptable (server is alive); reports TLS cert details |
| `internal/protocols/ews/testauth.go` | GetFolder(Inbox) to verify credentials; reports auth method used |
| `internal/protocols/ews/getfolder.go` | GetFolder(Inbox) with full output: display name, total/unread count, folder ID |
| `internal/protocols/ews/autodiscover.go` | POST to `/autodiscover/autodiscover.svc`; reports resolved EWS URLs |

### `config.go` — Config struct fields

```
Host             string          // Exchange server hostname
Port             int             // default 443
EWSPath          string          // default /EWS/Exchange.asmx
AutodiscoverPath string          // default /autodiscover/autodiscover.svc
Timeout          time.Duration   // default 30s
Username         string          // DOMAIN\user for NTLM, email for Basic/Bearer
Password         string
AccessToken      string          // OAuth2 Bearer token
AuthMethod       string          // NTLM, Basic, Bearer, auto
Domain           string          // AD domain (extracted from username if DOMAIN\user)
Mailbox          string          // Target mailbox for impersonation (optional)
SkipVerify       bool
TLSVersion       string          // 1.2, 1.3
ProxyURL         string
VerboseMode      bool
LogLevel         string
LogFormat        string          // csv, json
```

**`auto` auth resolution in `validateConfiguration`:**
- `AccessToken` set → `Bearer`
- `Username` contains `\` or `Domain` is set → `NTLM`
- Otherwise → `Basic`

**Action-specific validation:**
- `testauth`, `getfolder`: require `Username` + (`Password` or `AccessToken`)
- `autodiscover`: require `Username` in email format (validated via `validation.ValidateEmail`)

### `cmd.go` — subcommand pattern (mirrors `smtp/cmd.go` exactly)

```go
func NewCmd() *cobra.Command {
    v := viper.New()
    cmd := &cobra.Command{Use: "ews", Short: "EWS (Exchange Web Services) testing"}
    RegisterPersistentFlags(cmd)
    BindEnvs(v)
    cmd.AddCommand(
        newTestConnectCmd(v),
        newTestAuthCmd(v),
        newGetFolderCmd(v),
        newAutodiscoverCmd(v),
    )
    return cmd
}
```

Each subcommand RunE:
1. `v.BindPFlags(cmd.Flags())` + `v.BindPFlags(cmd.InheritedFlags())`
2. `ConfigFromViper(v)` → set `config.Action`
3. `validateConfiguration(config)`
4. `bootstrap.SetupSignalContext()`
5. `bootstrap.InitLoggers("ewstool", action, ...)`
6. Call action handler (`testConnect`, `testAuth`, `getFolder`, `autodiscover`)

## Usage Examples

```bash
# Connectivity check — no credentials needed
gomailtest ews testconnect --host mail.company.com
gomailtest ews testconnect --host mail.company.com --skipverify

# Auth test — NTLM (auto-detected from DOMAIN\user format)
gomailtest ews testauth --host mail.company.com --username "CORP\jsmith" --password "..."

# Auth test — Bearer token (Exchange Online EWS)
gomailtest ews testauth --host outlook.office365.com --username user@company.com --accesstoken "..."

# Get Inbox folder info
gomailtest ews getfolder --host mail.company.com --username "CORP\jsmith" --password "..."

# Autodiscover
gomailtest ews autodiscover --host mail.company.com --username user@company.com --password "..."
```
