# v3.3.4 — NTLM Authentication for SMTP

## What Changed

NTLM (NTLMv2) is now a supported authentication method for the `gomailtest smtp` commands (`testauth`, `sendmail`). This enables testing on-premises Exchange and Windows SMTP servers that require Windows Integrated Authentication.

No new dependency was introduced — `github.com/Azure/go-ntlmssp` was already used by `ewstool`.

## Username Format

NTLM usernames may include the Active Directory domain in either of two formats:

| Format | Example | Result |
|--------|---------|--------|
| `DOMAIN\user` | `CONTOSO\alice` | domain = `CONTOSO`, user = `alice` |
| plain user | `alice` or `alice@contoso.com` | no domain prefix sent |

The domain is parsed automatically from the username — no separate `--domain` flag is required.

## Auto-Detection Priority Change

When `--authmethod auto` (default), the mechanism preference order is now:

**Without access token:** `CRAM-MD5` → **`NTLM`** → `PLAIN` → `LOGIN`

**With `--accesstoken`:** `XOAUTH2` → `CRAM-MD5` → **`NTLM`** → `PLAIN` → `LOGIN`

NTLM will only be selected if the server explicitly advertises it in the EHLO `AUTH` capability list.

## Example Usage

```powershell
# Explicit NTLM with domain-prefixed username
gomailtest smtp testauth --host exchange.contoso.com --port 25 \
  --username "CONTOSO\alice" --password "secret" --authmethod NTLM

# Auto-detect (picks NTLM if server advertises it, falls back otherwise)
gomailtest smtp testauth --host exchange.contoso.com --port 587 \
  --username "CONTOSO\alice" --password "secret"

# Send mail via NTLM-authenticated relay
gomailtest smtp sendmail --host exchange.contoso.com --port 25 \
  --username "CONTOSO\alice" --password "secret" \
  --from alice@contoso.com --to bob@contoso.com \
  --subject "Test" --body "NTLM relay test"
```
