# Version 3.0.5

## Changes

### Units D.1–D.4 — Port remaining protocols to Cobra (3.0 migration)

All four remaining protocols (`smtp`, `pop3`, `imap`, `jmap`) ported from the legacy flag-based tool binaries to Cobra subcommands under `gomailtest`. Pattern follows Unit C (msgraph).

**New packages:**

- `internal/protocols/smtp/` — `testconnect`, `teststarttls`, `testauth`, `sendmail` subcommands; `sendmail`-only flags (`--from`, `--to`, `--subject`, `--body`) registered locally, not as persistent flags
- `internal/protocols/pop3/` — `testconnect`, `testauth`, `listmail` subcommands; `--maxmessages` flag local to `listmail`
- `internal/protocols/imap/` — `testconnect`, `testauth`, `listfolders` subcommands; full IMAPS/STARTTLS mutual exclusion validation and smart port default (143→993 when `--imaps` set)
- `internal/protocols/jmap/` — `testconnect`, `testauth`, `getmailboxes` subcommands; HTTPS with Bearer/Basic auth, connect-address override for load balancer testing

**Backward-compat shims** replaced in `cmd/smtptool/`, `cmd/pop3tool/`, `cmd/imaptool/`, `cmd/jmaptool/` — each translates old `-action=X -flag=Y` style to `gomailtest <proto> X --flag=Y`.

**Root registration:** all four `NewCmd()` registered in `cmd/gomailtest/root.go`.

### Unit E — Consolidate READMEs (3.0 migration)

Replaced 5 tool-specific root-level READMEs with a unified `docs/protocols/` structure.

**Deleted:** `SMTP_TOOL_README.md`, `IMAPTOOL_README.md`, `POP3TOOL_README.md`, `JMAPTOOL_README.md`, `MSGRAPHTOOL_README.md`

**New:** `docs/protocols/{smtp,imap,pop3,jmap,msgraph}.md` — per-protocol docs with updated `gomailtest <proto> <action> --flag` command style and legacy shim migration notes

**Updated:** `README.md` (full rewrite, all 5 protocols, commands table, env var prefix table), `BUILD.md` (updated project tree, build commands), `EXAMPLES.md` (all msgraph examples updated to new style), `TROUBLESHOOTING.md` (condensed rewrite), `SECURITY.md` (updated tool invocations), `UNIT_TESTS.md` (removed `src/` references, updated to `go test ./...`), `INTEGRATION_TESTS.md` (new command style, cross-protocol examples)
