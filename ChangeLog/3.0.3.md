# Version 3.0.3

## Changes

### Unit C — Cobra skeleton + Microsoft Graph port (3.0 migration)

Introduced the unified `gomailtest` binary with Cobra/Viper command routing, and ported `msgraphtool` as the first protocol.

**New binary:** `gomailtest` — single entry point for all protocols. Replaces the five individual tool binaries over the 3.0.x release series.

**New package `internal/protocols/msgraph`:**

All logic moved from `cmd/msgraphtool/` and restructured for Cobra:
- `cmd.go` — `NewCmd()` exposing `msgraph` Cobra command with 7 subcommand leaves: `getevents`, `sendmail`, `sendinvite`, `getinbox`, `getschedule`, `exportinbox`, `searchandexport`
- `config.go` — `Config` struct, `NewConfig()`, Viper-based `RegisterPersistentFlags()`, `BindEnvs()`, `ConfigFromViper()`
- `auth.go`, `cert_stub.go`, `cert_windows.go`, `utils.go`, `handlers.go` — moved, package renamed from `main` to `msgraph`

**New `cmd/gomailtest/`:**
- `main.go` — 10-line entry point calling `Execute()`
- `root.go` — root `gomailtest` Cobra command; registers `msgraph.NewCmd()`

**Backward-compat shim:** `cmd/msgraphtool/main.go` replaced with exec-based shim that translates old `-action=X -flag=Y` style to `gomailtest msgraph X --flag=Y`.

**Key design decisions:**
- Viper instance is per-protocol (created in `NewCmd()`) — prevents flag name collisions between protocols
- Signal context per subcommand `RunE` — keeps each action self-contained
- `--verbose` on `msgraph` persistent flags, not root — other protocols may have different verbosity
