# Version 3.0.2

## Changes

### Unit B — Shared bootstrap package (3.0 migration)

Extracted the identical startup boilerplate from all 5 `cmd/*tool/main.go` files into a new `internal/common/bootstrap/` package. Pure refactor, no user-visible change beyond one standardized log message.

**New package `internal/common/bootstrap`:**

- `SetupSignalContext() (context.Context, context.CancelFunc)` — cancellable context wired to SIGINT/SIGTERM. Replaces the duplicated signal-handling block in every main.
- `InitLoggers(toolName, action string, verbose bool, logLevel, logFormat string) (*slog.Logger, logger.Logger, error)` — wraps `logger.SetupLogger` + `logger.ParseLogFormat` + `logger.NewLogger` in one call.

**Refactored:** `cmd/{smtptool,imaptool,pop3tool,jmaptool,msgraphtool}/main.go` now delegate to the bootstrap helpers. `msgraphtool`'s lenient "warn and continue on file-logger failure" behavior is preserved. Each `main.go` shrinks by 25–30 lines.

### Minor behavior change

The shutdown message printed on Ctrl+C by `imaptool`, `pop3tool`, and `jmaptool` is now standardized to match `smtptool`/`msgraphtool`:

- Before: `Received interrupt signal, shutting down...`
- After: `Received interrupt signal. Shutting down gracefully...`

No scripted callers should depend on this text.
