# Version 2.7.0

## Changes

### Connection Address Override for All Mail Tools

Extended the `-address` flag feature (originally added to smtptool in v2.6.19) to all mail testing tools:

- **imaptool**: Add `-address` flag with `IMAPADDRESS` environment variable
- **pop3tool**: Add `-address` flag with `POP3ADDRESS` environment variable
- **jmaptool**: Add `-address` flag with `JMAPADDRESS` environment variable

This feature allows connecting to a specific IP address or hostname while using the original `-host` value for TLS Server Name Indication (SNI) and certificate verification. Useful for testing servers behind load balancers or DNS round-robin configurations.

**Example:**
```bash
# Connect to specific IP while using hostname for TLS verification
imaptool -action testconnect -host imap.example.com -address 192.168.1.10 -imaps
pop3tool -action testconnect -host pop.example.com -address 10.0.0.5 -pop3s
jmaptool -action testconnect -host jmap.example.com -address 172.16.0.1
```

### Bug Fixes

- **IPv6 Support**: Fixed address formatting for imaptool and pop3tool to properly support IPv6 addresses using `net.JoinHostPort()`
- **jmaptool Performance**: Optimized HTTP transport by reusing `net.Dialer` instance instead of allocating on each connection

### Documentation

- Updated all tool README files with `-address` flag documentation
- Added `{PREFIX}ADDRESS` to common environment variables in TOOLS.md
