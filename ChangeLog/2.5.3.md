# Change Log: Version 2.5.3

**Release Date:** 2026-01-30

This patch release fixes security logging gaps and improves validation for XOAUTH2 authentication.

## Bug Fixes

### Security Logging Improvements

- **Fixed access token masking in error logs** - `maskAccessToken()` is now used in error logging for both `testauth` and `sendmail` actions, ensuring OAuth2 access tokens are properly masked when authentication fails

### Validation Improvements

- **Added warning for conflicting credentials** - When both `-password` and `-accesstoken` are provided, the tool now displays a warning message indicating that the password will be ignored in favor of XOAUTH2 authentication

### Documentation

- **Updated changelog accuracy** - Corrected the 2.5.0 changelog to accurately reflect that `maskAccessToken()` is used in error logging
